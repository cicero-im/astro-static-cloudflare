# Paraglide Message Compilation

Compile `messages/<locale>.json` files into typed JS modules that pages import directly. Unused locales are tree-shaken from each page's bundle.

## Requirements

- `project.inlang/settings.json` declares `baseLocale`, `locales[]`, and the message-format plugin
- Translation files live at `messages/<locale>.json` (one per locale)
- The compiled output (`src/paraglide/`) is git-ignored — regenerated by the dev server, the build, or `bun run paraglide:compile`
- A middleware reads the request's locale and calls `setLocale()` so server-rendered pages emit the correct strings

## Implementation Pattern

Configure `paraglideVitePlugin` in `astro.config.mjs` with:

- `strategy: ["url", "cookie", "baseLocale"]` — resolution order: URL prefix wins, falls back to a sticky cookie, then to `baseLocale`
- `outputStructure: "message-modules"` — one module per message so dead-code elimination can drop unused locales
- `cookieName` for explicit locale switching

Astro's built-in `i18n.routing.prefixDefaultLocale: false` keeps `baseLocale` URLs unprefixed; every other locale is prefixed (`/pt/...`, `/es/...`).

## Adding a Locale

1. Add the BCP-47 mapping to `LOCALE_LANGUAGE_TAGS` in `astro.config.mjs`
2. Add the locale tag to `locales[]` in `project.inlang/settings.json`
3. Create `messages/<locale>.json` (or run `bun run i18n:translate` to machine-translate)
4. Restart the dev server so Paraglide regenerates `src/paraglide/`
